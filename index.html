프로젝트 소개

학부모·학생·선생님을 하나로 연결하는 축구단 전용 등하원·출결 관리 솔루션 — 백그라운드 GPS 실시간 추적, QR 간편 출결, FCM 푸시 알림을 크로스플랫폼 앱과 관리자 웹으로 통합하고, 기존 CRM과 연동하며 SaaS 확장형 멀티테넌트 구조로 설계합니다.

프로젝트 개요

항목내용

핵심 요구사항 분석

  • 회원 및 권한 관리 (기존 CRM 연동) — 자체 제작 CRM DB 연동 API 개발 및 회원 정보 동기화, 학부모·학생·선생님 3권한별 로그인 및 맞춤형 홈 라우팅, 추후 SaaS 확장을 고려한 Tenant ID 기반 DB 설계(기능 구현 제외)
  • 차량 운행 및 실시간 위치 추적 — 선생님용 운행 시작/종료 제어, 앱 백그라운드 GPS 실시간 송신, 학생·학부모용 운행 차량 실시간 지도 표시, 운행 종료 시 송신 자동 차단 및 운행 기록 저장
  • 출결 및 픽업 관리 — 차량 내 QR 스캔 간편 출석, 선생님용 탑승/하차 수동 체크, 학부모용 탑승·하차·도착 FCM 푸시 알림, 일자별·학생별 출결 히스토리 조회
  • 커뮤니티 및 관리자 웹 — 학부모 공지/게시판(글쓰기·댓글), 관리자 웹 출결 통계 대시보드, 차량 픽업 노선·정류장 관리, 커뮤니티 모니터링 및 선수 기본 정보 관리
  • 비기능 요구사항 — iOS·Android 백그라운드 환경 안정적 GPS 송신 보장, 학생 개인정보·위치 정보 보호를 위한 암호화 통신 적용

제안 포인트

1. 출결·CRM·예약형 운영 앱을 3플랫폼(Android·iOS·웹)으로 동시 출시한 경험을 그대로 적용합니다. 회원·강사·관리자 3종 권한, 출결 관리, CRM, 알림을 하나의 앱과 관리자 웹으로 통합 출시한 경험이 본 프로젝트의 학부모·학생·선생님 3권한 구조와 직접 대응됩니다.

2. 백그라운드 실시간 위치·지도 기능을 안정적으로 구현합니다. 모바일 근거리 탐색·실시간 위치 동기화 및 Firebase Realtime DB 기반 즉시 동기화 구현 경험을 바탕으로, iOS·Android 백그라운드 GPS 송신과 배터리·권한 정책에 맞춘 안정적 위치 추적을 설계합니다.

3. SaaS 확장을 처음부터 고려한 멀티테넌트 DB를 설계합니다. 멀티테넌트 아키텍처로 복수 시설을 단일 시스템에서 운영한 경험을 살려, 1단계 자사 도입에 최적화하되 Tenant ID 기반 테이블 구조로 추후 타 기관 확장 시 스키마 재설계가 불필요하도록 ERD를 구성합니다.

4. QR 출결과 FCM 푸시를 묶어 보호자 경험을 완성합니다. QR 스캔 기반 즉시 처리와 7종 이상의 상황별 FCM 푸시 + 딥링크 구현 경험으로, 탑승·하차·도착 시점의 실시간 알림 흐름을 누락 없이 구성합니다.

5. 기획 공수를 제외한 디자인·개발 중심 견적으로 산정했습니다. 제공되는 스토리보드(요구사항 정의서·화면설계서·IA)를 기준으로 디자인과 개발에 집중하여, 다수 인력의 디자인↔개발↔백엔드 병행으로 60일 일정을 준수합니다.

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기능 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

축구단을 운영 중인 업체가 학부모·학생·선생님이 모두 사용하는 전용 등하원 관리 앱을 구축합니다. 기존 자체 제작 CRM 및 회원 DB와 연동하며, 현재는 자사 축구단 전용으로 사용하되 추후 SaaS 방식으로 타 기관에 서비스할 수 있도록 확장성을 고려한 DB 아키텍처를 설계하는 것이 핵심 목표입니다. 내부 전산팀이 작성한 기획서(스토리보드)를 기준으로 디자인 및 개발을 수행합니다.

  • [필수] 기존 자체 제작 CRM DB 연동 API 및 회원 정보 동기화
  • [필수] 학부모·학생·선생님 3권한별 로그인 및 맞춤형 홈 화면 라우팅
  • [필수] Tenant ID 기반 멀티테넌트 DB 테이블 설계 (멀티테넌트 기능 구현은 범위 외)
  • [필수] 비밀번호 찾기, 프로필 수정, 앱 내 푸시 알림 수신 동의 설정

3.2 차량 운행 및 실시간 위치 추적

  • [필수] 선생님용 운행 시작/종료 버튼 제어
  • [필수] 앱 백그라운드 상태에서의 GPS 위치 실시간 송신
  • [필수] 학생·학부모용 운행 차량 실시간 위치 지도 표시
  • [필수] 운행 종료 시 위치 송신 자동 차단 및 운행 기록 저장

3.3 출결 및 픽업 관리

  • [필수] 학생용 차량 내 QR 코드 스캔 간편 출석 체크
  • [필수] 선생님용 학생 탑승·하차 수동 체크
  • [필수] 학부모용 자녀 탑승·하차·학원 도착 시 실시간 FCM 푸시 알림
  • [필수] 일자별·학생별 출결 히스토리 조회

3.4 커뮤니티 및 관리자 웹

  • [필수] 학부모 커뮤니티 공지 확인 및 게시판 글쓰기/댓글
  • [필수] 관리자 웹 전체 학생 출결 통계 대시보드
  • [필수] 관리자 웹 차량 픽업 노선 관리 및 정류장 세팅
  • [필수] 관리자 웹 커뮤니티 게시글 모니터링 및 선수 기본 정보 관리

5. 기술 제약사항

프론트엔드(App)는 React Native, 프론트엔드(Web)는 React.js, 백엔드는 Node.js를 사용합니다. 데이터베이스는 MySQL을 기본으로 하며 실시간 위치 추적에는 Firebase Realtime DB를 활용합니다. 인프라는 기존 AWS 환경을 활용합니다. 기존 자체 제작 CRM의 DB 스키마/연동 방식은 착수 시 제공받아 연동 API를 설계합니다.
